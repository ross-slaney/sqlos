SqlOS is auth and authorization for .NET in one package. Identity, SSO, OIDC, and FGA run in your app. Data stays in **your** SQL Server.

## Why SqlOS

Many auth products host your users and charge per seat. SqlOS does not:

- **Your database** -- users, sessions, grants, and tokens live in your SQL Server instance
- **Your app** -- SqlOS runs inside your ASP.NET process, not as a separate service
- **One package** -- `dotnet add package SqlOS` and you're done
- **Standards-based** -- OAuth 2.0, PKCE, JWKS, SAML, OpenID Connect, `CIMD`, `DCR`, and resource indicators when you need them

Setup stays the same:

```csharp
builder.AddSqlOS<AppDbContext>();
```

FGA filters data in SQL so you do not load rows you should not see.

```csharp
var chains = await dbContext.Chains
    .Where(await authService.GetAuthorizationFilterAsync<Chain>(subjectId, "CHAIN_VIEW"))
    .ToListAsync();
```
