**What this guide covers**: Session idle timeout, absolute lifetime, refresh token lifetime, dashboard configuration.

**Dashboard path**: Auth Server → Security  
**URL**: `http://localhost:5062/sqlos/admin/auth/security`

---

Security settings control how long sessions and tokens remain valid. They can be configured via the dashboard or the admin API.

## Settings

| Setting | Description | API field |
|---------|-------------|-----------|
| **Refresh token lifetime** | How long a refresh token is valid | `refreshTokenLifetimeMinutes` |
| **Session idle timeout** | Session expires after this period of inactivity | `sessionIdleTimeoutMinutes` |
| **Session absolute lifetime** | Hard session expiration regardless of activity | `sessionAbsoluteLifetimeMinutes` |

## Dashboard

The Security page shows current values and allows updating them.

## API

```text
GET  /sqlos/admin/auth/api/settings/security
PUT  /sqlos/admin/auth/api/settings/security
```

```json
{
  "refreshTokenLifetimeMinutes": 10080,
  "sessionIdleTimeoutMinutes": 1440,
  "sessionAbsoluteLifetimeMinutes": 43200
}
```

## Code Example

Security settings are resolved at runtime by `SqlOSSettingsService`:

```csharp
var settings = await settingsService.GetResolvedSecuritySettingsAsync(cancellationToken);
// settings.RefreshTokenLifetime    → TimeSpan
// settings.SessionIdleTimeout      → TimeSpan
// settings.SessionAbsoluteLifetime → TimeSpan
```

These values are used internally when creating sessions and refresh tokens. The access token lifetime is configured separately in `SqlOSAuthServerOptions.AccessTokenLifetime`.
