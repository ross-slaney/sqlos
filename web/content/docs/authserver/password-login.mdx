# Password Login

**What this guide covers**: Password login flow, multi-organization selection, pending auth tokens.

---

Password login authenticates a user by email and password. If the user belongs to multiple organizations, the flow pauses for organization selection.

## Flow

1. Client calls `LoginWithPasswordAsync` with email, password, and client ID
2. Service validates credentials against the stored password hash
3. If user has multiple org memberships and no `organizationId` was provided → returns `RequiresOrganizationSelection = true` with a `PendingAuthToken`
4. Client calls `SelectOrganizationAsync` with the pending token and chosen org
5. Session and tokens are created

## Code Example

```csharp
var result = await authService.LoginWithPasswordAsync(
    new SqlOSPasswordLoginRequest(email, password, clientId, organizationId: null),
    httpContext,
    cancellationToken);

if (result.RequiresOrganizationSelection)
{
    // Present result.Organizations to user, then:
    var tokens = await authService.SelectOrganizationAsync(
        new SqlOSSelectOrganizationRequest(result.PendingAuthToken, selectedOrgId),
        httpContext,
        cancellationToken);
}
else
{
    var tokens = result.Tokens;
    // tokens.AccessToken, tokens.RefreshToken, tokens.SessionId
}
```

## Configuration

Password auth can be disabled via `SqlOSAuthServerOptions.EnableLocalPasswordAuth`. When disabled, `LoginWithPasswordAsync` throws.

Verified email can be required via `RequireVerifiedEmailForPasswordLogin`.
