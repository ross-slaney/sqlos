# Mutation / Create Resource

**What this guide covers**: Authorization check before mutation, creating a resource in the hierarchy.

---

For create/update/delete operations, check authorization first with `CheckAccessAsync`, then create the FGA resource alongside your domain entity.

## Full Create Flow

From the retail example — creating a chain:

```csharp
group.MapPost("/", async (
    CreateChainRequest request,
    ExampleAppDbContext context,
    ISqlOSFgaAuthService authService,
    HttpContext http) =>
{
    var subjectId = http.GetSubjectId();

    // 1. Check permission on the parent resource
    var access = await authService.CheckAccessAsync(
        subjectId, RetailPermissionKeys.ChainEdit, "retail_root");
    if (!access.Allowed)
        return Results.Json(new { error = "Permission denied" }, statusCode: 403);

    // 2. Create FGA resource as child of parent
    var resourceId = context.CreateResource(
        "retail_root", request.Name, RetailResourceTypeIds.Chain);

    // 3. Create domain entity with the resource ID
    var chain = new Chain
    {
        ResourceId = resourceId,
        Name = request.Name,
        Description = request.Description,
        HeadquartersAddress = request.HeadquartersAddress
    };
    context.Chains.Add(chain);

    // 4. Save both in one transaction
    await context.SaveChangesAsync();

    return Results.Created($"/api/chains/{chain.Id}", new ChainDetailDto { ... });
});
```

## Pattern Summary

1. **Check** — `CheckAccessAsync` on the parent resource with the edit permission
2. **Create resource** — `context.CreateResource(parentId, name, typeId)` returns a resource ID
3. **Create entity** — assign `ResourceId` to your domain model
4. **Save** — both the FGA resource and domain entity save in one `SaveChangesAsync` call
