**What this guide covers**: `CheckAccessAsync` for point authorization checks.

---

Use `CheckAccessAsync` when you need a yes/no answer: can this subject perform this action on this resource?

## Code Example

From the retail example — checking before creating a chain:

```csharp
var access = await authService.CheckAccessAsync(
    subjectId,
    RetailPermissionKeys.ChainEdit,  // "CHAIN_EDIT"
    "retail_root"                     // resource ID
);

if (!access.Allowed)
    return Results.Json(new { error = "Permission denied" }, statusCode: 403);
```

## Method Signature

```csharp
Task<SqlOSFgaAccessCheckResult> CheckAccessAsync(
    string subjectId,
    string permissionKey,
    string resourceId);
```

## Result

| Field | Type | Description |
|-------|------|-------------|
| `Allowed` | `bool` | Whether access is granted |

## How It Works

1. Find all grants for the subject
2. For each grant, check if the grant's role includes the required permission
3. Walk up the resource hierarchy from `resourceId` to root
4. If any grant on any ancestor matches → `Allowed = true`

## When to Use

- **Mutations** (create, update, delete) — check before writing
- **Single-resource access** where you already have the resource ID
- For **list queries**, prefer `GetAuthorizationFilterAsync` instead
